// Architecture IPBX — Téléphonie IP d'entreprise

IPBX (IP Private Branch eXchange) est un autocommutateur téléphonique privé fonctionnant sur protocole SIP/H.323 via le réseau IP. Il remplace l'ancien PABX analogique et gère toutes les communications internes/externes de l'entreprise. Orange propose des solutions 3CX, Alcatel OXO Connect, ou Business Talk (cloud). Les flux vocaux sont chiffrés via SRTP et la signalisation via TLS.

Analyse réflexive & autoévaluation
Ce que j'ai appris
L'IPBX est ma mission la plus transversale : elle mêle réseau (VLAN, QoS, sécurité périmétrique), télécoms (codecs, SIP, RTP) et système (administration serveur 3CX/Asterisk). J'ai compris pourquoi la voix sur IP est si sensible à la qualité réseau : un seul paquet RTP perdu sur 200 = grésillement audible. La QoS ne peut pas être en option.
Difficultés rencontrées
Lors d'une migration PABX analogique → IPBX 3CX, j'ai sous-estimé la complexité du portage des numéros (SDA) et du mapping des extensions historiques. Résultat : des utilisateurs n'ont pas pu être joints pendant une demi-journée. C'est ma plus grande erreur jusqu'à présent et l'impact business était lourd.
Comment je les ai surmontées
J'ai construit avec mon tuteur une grille de checklist de bascule téléphonie (extensions, SDA, renvois actifs, groupes d'appel, plages horaires) qui est désormais utilisée par l'équipe. J'ai aussi appris l'analyse de flux SIP avec Wireshark pour diagnostiquer les problèmes d'enregistrement (REGISTER refusé, codec mismatch, etc.).
Évolution de mon autonomie
Je gère seul l'ajout/suppression de postes IP sur un IPBX existant. Sur les déploiements complets ou migrations PABX→IPBX, je travaille toujours en binôme avec mon tuteur — l'enjeu d'une coupure téléphonique en environnement de production reste trop sensible pour que j'opère seul.
Niveau auto-évalué : Intermédiaire Axes de progrès : approfondir Asterisk en ligne de commande (dialplan custom), comprendre les attaques toll-fraud et anti-spam SIP.
NOTICE D'INSTALLATION

Fiche Technique — IPBX

Prérequis
Matériel
  • Serveur ou appliance IPBX (3CX / Alcatel OXO Connect)
  • Téléphones IP compatibles SIP (PoE)
  • Switch PoE+ dédié voix (VLAN voix séparé)
  • Base et combinés DECT si déploiement sans fil
  • Câbles RJ45 Cat6, alimentation ondulée (UPS)
Informations client
  • Plan de numérotation interne (plage d'extensions)
  • Numéros DDI (Direct Dial In) fournis par Orange
  • Paramètres du trunk SIP Orange (serveur, credentials)
  • Liste des utilisateurs et de leurs besoins (renvoi, groupe…)
Étapes d'installation
1
Analyse des besoins et plan de numérotation

Recenser le nombre de postes internes, les groupements d'appels, les files d'attente, l'IVR (accueil vocal), les renvois et les numéros DDI. Créer un plan de numérotation (ex : 1xx pour postes, 200-299 pour salles, 300+ pour groupements). Valider avec le client avant installation.

2
Installation physique du serveur IPBX

Installer le serveur dans la baie réseau du client. Connecter sur le VLAN voix dédié. Pour une solution virtualisée (3CX sur VM), créer la VM avec les ressources recommandées (4 vCPU, 8 Go RAM minimum pour > 50 postes). Mettre à jour le firmware/OS avant configuration.

Le réseau VoIP doit obligatoirement être sur un VLAN dédié, séparé du VLAN données, pour garantir la qualité audio et la sécurité.
3
Configuration du trunk SIP Orange

Créer le trunk SIP dans l'interface admin de l'IPBX. Renseigner le serveur SIP d'Orange (proxy SIP), les credentials d'authentification, les numéros DDI. Sélectionner les codecs dans l'ordre de priorité : G.711 ulaw/alaw, puis G.729 pour économiser la bande passante. Activer SIP OPTIONS heartbeat toutes les 30s.

4
Création des extensions et groupements

Créer chaque extension SIP avec : numéro interne, nom, mot de passe SIP fort. Créer les groupements d'appels (ring all / round-robin / least-used). Configurer les files d'attente avec musique d'attente, annonces et time-out. Créer l'IVR (menu vocal) avec les routes selon les touches.

5
Auto-provisioning des téléphones IP

Configurer l'option DHCP 66 (TFTP server) pointant vers le serveur de provisioning de l'IPBX. Brancher les téléphones sur les ports PoE : ils récupèrent automatiquement leur configuration (adresse IP, serveur SIP, extension). Vérifier l'enregistrement SIP de chaque poste dans la console IPBX.

Si le DHCP option 66 n'est pas disponible, utiliser le PnP (Plug and Play) via URL de configuration encodée dans le téléphone ou la provisioning platform du fabricant.
6
Sécurisation de l'IPBX

Installer et configurer fail2ban (bloquer après 5 échecs SIP REGISTER). Activer TLS pour la signalisation SIP (port 5061) avec un certificat valide. Activer SRTP pour le chiffrement des flux RTP. Restreindre l'accès admin à l'IP du réseau LAN uniquement. Désactiver les comptes SIP par défaut.

Un IPBX non sécurisé est une cible prioritaire pour les fraudeurs téléphoniques (toll fraud). Une seule compromission peut générer des milliers d'euros de communications vers des numéros surtaxés.
7
QoS VoIP et tests de qualité

Configurer le marquage DSCP EF (Expedited Forwarding, valeur 46) sur les flux RTP dans le switch et le routeur. Vérifier la priorisation sur toute la chaîne réseau. Tester la qualité audio avec le score MOS (Mean Opinion Score) > 4 attendu. Effectuer des tests de charge (appels simultanés).

Checklist de validation
Trunk SIP Orange enregistré et actif
Tous les postes SIP enregistrés
Appels internes et externes testés
IVR et messagerie vocale fonctionnels
SRTP/TLS actifs (chiffrement vocal)
fail2ban actif (anti-brute force SIP)
QoS DSCP EF configurée sur le réseau
Formation utilisateurs effectuée

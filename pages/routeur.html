Présentation de la prestation

Installation et configuration de routeur professionnel

La prestation d'installation de routeur consiste à déployer physiquement et configurer logiquement un routeur professionnel sur le site client. Ce routeur constitue la pièce centrale de l'infrastructure réseau : il assure la connexion WAN vers Orange Business, le routage des flux entre les différents segments du réseau local (LAN), et la sécurité périmétrique via les ACL et le NAT.

Orange Business déploie principalement des routeurs Cisco ISR 4000/4300, Huawei AR6000 et Juniper SRX selon les contrats et les besoins en performances. Chaque installation suit un processus standardisé garantissant la conformité, la sécurité et la traçabilité.

  • Montage physique en baie 19" ou en table (selon locaux)
  • Configuration CLI complète via port console
  • Mise en place du routage statique ou dynamique (OSPF/BGP)
  • Sécurisation : ACL, SSH, désactivation services inutiles
  • Intégration au monitoring SNMP Orange
  • Documentation complète et remise en main client

Notice d'installation — Cisco IOS
01

Montage physique en baie 19"

Installer le routeur dans la baie selon le plan d'implantation. Respecter les règles de gestion thermique (flux d'air avant → arrière).

Étapes de montage
✓ Fixer les oreilles de rack (kit inclus avec le routeur) ✓ Insérer en baie — position U selon plan d'implantation ✓ Câbler l'alimentation secteur (redondée si 2 PSU) ✓ Passer les câbles dans les guides-câbles latéraux ✓ Étiqueter chaque câble : SIDE-A WAN / LAN-1 / CONSOLE ✓ Brancher le câble console RJ45 → laptop technicien
02

Configuration initiale via console série

Premier accès au routeur via PuTTY en mode console. Configurer les éléments de base de sécurité et d'identification.

PuTTY : 115200 baud / 8-N-1 / No flow control
Router> enable Router# configure terminal ! Identification hostname RTR-CLIENT-PARIS01 ip domain-name client.local ! Sécurité accès enable secret MotDePasseFort!2024 service password-encryption no ip http server no ip http secure-server ! Console + VTY line console 0 logging synchronous exec-timeout 10 0 line vty 0 4 transport input ssh login local
03

Configuration interface WAN

Configurer le lien vers Orange Business (IP statique fournie ou PPPoE selon l'offre souscrite).

Interface WAN — IP statique (mode Ethernet)
interface GigabitEthernet0/0/0 description WAN-ORANGE-BUSINESS ip address 90.85.120.14 255.255.255.252 ip nat outside no shutdown ! Route par défaut vers passerelle Orange ip route 0.0.0.0 0.0.0.0 90.85.120.13 ! Vérification show interface GigabitEthernet0/0/0 → GigabitEthernet0/0/0 is up, line protocol is up
04

Configuration des interfaces LAN (sous-interfaces VLAN)

Créer les sous-interfaces avec encapsulation 802.1Q pour chaque VLAN du client. Le trunk vers le switch est sur une interface physique unique.

Sous-interfaces VLAN — Trunk vers switch
interface GigabitEthernet0/0/1 no ip address no shutdown ! VLAN 10 — DATA interface GigabitEthernet0/0/1.10 encapsulation dot1Q 10 ip address 192.168.10.1 255.255.255.0 ip nat inside description VLAN-DATA-POSTES ip helper-address 192.168.10.10 <!-- DHCP server si externe --> ! VLAN 20 — VOIX (avec QoS) interface GigabitEthernet0/0/1.20 encapsulation dot1Q 20 ip address 10.0.20.1 255.255.255.0 ip nat inside description VLAN-VOIX-IPBX ! VLAN 99 — MANAGEMENT interface GigabitEthernet0/0/1.99 encapsulation dot1Q 99 native ip address 172.16.99.1 255.255.255.0 description VLAN-MGMT
05

Configuration OSPF et routage dynamique

Activer OSPF pour les déploiements multi-sites. Sur site unique, le routage statique suffit.

OSPF Area 0 — Authentification MD5
router ospf 1 router-id 1.1.1.1 network 192.168.10.0 0.0.0.255 area 0 network 10.0.20.0 0.0.0.255 area 0 network 172.16.99.0 0.0.0.255 area 0 passive-interface GigabitEthernet0/0/0 default-information originate ! Authentification MD5 sur interface interface GigabitEthernet0/0/1.10 ip ospf authentication message-digest ip ospf message-digest-key 1 md5 OspfKey2024! ! Vérification voisins OSPF show ip ospf neighbor show ip route ospf
06

ACL de filtrage et sécurité périmétrique

Configurer les listes de contrôle d'accès pour bloquer le trafic indésirable en entrée du WAN et contrôler les flux inter-VLAN.

ACL étendue — Protection WAN entrant
ip access-list extended BLOCK-WAN-IN ! Bloquer RFC1918 en entrée (anti-spoofing) deny ip 10.0.0.0 0.255.255.255 any deny ip 172.16.0.0 0.15.255.255 any deny ip 192.168.0.0 0.0.255.255 any ! Autoriser connexions établies permit tcp any any established ! Autoriser ICMP (ping de debug) permit icmp any any echo-reply permit icmp any any unreachable ! Bloquer tout le reste deny ip any any log ! Appliquer sur interface WAN en entrée interface GigabitEthernet0/0/0 ip access-group BLOCK-WAN-IN in
07

Configuration SNMP v3 pour monitoring Orange

Activer SNMP v3 (chiffré et authentifié) pour intégration dans le système de supervision Orange Business.

SNMP v3 — Sécurisé
! Créer vue SNMP snmp-server view ORANGE_VIEW iso included ! Groupe SNMP v3 AuthPriv snmp-server group ORANGE_GRP v3 priv read ORANGE_VIEW ! Utilisateur SNMP v3 snmp-server user orange_monitor ORANGE_GRP v3 auth sha AuthPass2024! priv aes 128 PrivPass2024! ! Traps vers NOC Orange snmp-server host 10.x.x.x version 3 priv orange_monitor snmp-server enable traps ! Vérification show snmp user
08

Sauvegarde et validation finale

Sauvegarder la configuration en mémoire non-volatile, effectuer tous les tests de validation et rédiger le compte-rendu.

Sauvegarde + tests finaux
! Sauvegarder en NVRAM copy running-config startup-config → [OK] 4872 bytes ! Backup sur TFTP Orange copy running-config tftp: Address of remote host: 10.x.x.1 Destination filename: RTR-CLIENT-PARIS01-2024.cfg ! Tests finaux ping 8.8.8.8 → !!!!! OK ping 192.168.10.10 → !!!!! OK (VLAN 10) traceroute 8.8.8.8 → hop 1: 90.85.120.13 (GW Orange) show ip interface brief → all interfaces up show ip route → routes OK
Dépannage fréquent
Interface WAN down
Câble SFP mal inséré, ou lien physique côté Orange non actif
→ show interface Gi0/0/0 → Réinsérer le SFP → Contacter NOC Orange si lien non activé
OSPF voisin ne s'établit pas
Area ID différente, authentification MD5 incorrecte, ou hello/dead timer non concordants
→ debug ip ospf adj → Vérifier area 0 et clés MD5 identiques des deux côtés
VLAN non routé entre sous-interfaces
Interface physique parent sans "no shutdown", ou encapsulation dot1Q incorrecte
→ Vérifier "no shutdown" sur Gi0/0/1 parent → show interface Gi0/0/1.X
SSH refusé depuis LAN
Clés RSA non générées ou VTY line non configurée avec "transport input ssh"
→ crypto key generate rsa modulus 2048 → line vty transport input ssh

Installation de Routeur

Déploiement & configuration d'équipements réseau professionnels en baie

// Architecture — Routeur en baie 19"
Mission 2 / 5 · S1-S2 — Novembre 2025 à Mars 2026 · ~15 déploiements

Déploiement de routeurs professionnels (Cisco ISR, Huawei AR, Juniper SRX) chez les clients Orange Business. La prestation inclut le montage physique en baie 19", la configuration complète via console série, l'intégration dans l'infrastructure existante et les tests de validation en conditions réelles.

Contexte & objectifs
Type d'activité
Déploiement de routeurs en production sur sites clients PME/ETI : montage baie, config console, intégration dans architecture existante.
Enjeux techniques
Configuration OSPF/BGP sans rupture, ACL sécurisantes, hardening IOS, sauvegarde des configs avant toute modification.
Enjeux organisationnels
Intervention souvent en fenêtre de maintenance (soir/week-end) pour limiter la rupture de service. Coordination avec l'ingénieur réseau Orange.
Acteurs impliqués
Tuteur, ingénieur réseau Orange (préparation de la config cible), RSI client, équipe N3 si dépannage complexe.
Contraintes
Erreur de config = perte d'exploitation immédiate côté client. Gestion thermique en baie, étiquetage rigoureux exigé.
Objectif personnel
Maîtriser la syntaxe Cisco IOS et savoir déployer en autonomie une config OSPF mono-zone fournie par l'ingénieur.
Cisco IOSOSPFBGP ACLSSHSNMP v3SFP+

Sur les configurations BGP multi-AS ou redistribution complexe, je reste accompagné — l'enjeu d'erreur est trop élevé.

Analyse réflexive & autoévaluation
Ce que j'ai appris
L'installation en baie 19" m'a fait comprendre la différence entre le réseau « cours/TP » et le réseau réel : importance de l'étiquetage, de la documentation, de la gestion thermique, et du fait qu'une mauvaise config en production a un impact business immédiat (perte d'exploitation). J'ai vraiment intégré la chaîne câble console → mode privilégié → config → sauvegarde startup-config.
Difficultés rencontrées
La syntaxe Cisco IOS est dense et je me suis trompé plusieurs fois entre copy running-config startup-config et l'inverse, ce qui m'a coûté une demi-journée à reconfigurer un routeur après reboot. J'ai aussi eu du mal au début à lire les schémas réseau d'architecture cible (notation des sous-réseaux, conventions d'adressage internes Orange).
Comment je les ai surmontées
Je me suis créé une checklist personnelle (« commandes à ne jamais oublier ») et je sauvegarde systématiquement la config avant chaque modification avec copy running tftp:. Pour les schémas, j'ai pris l'habitude de redessiner sur papier les plages d'adresses avant toute config — ça a éliminé 90% de mes erreurs d'IP.
Évolution de mon autonomie
Du « je tape ce que mon tuteur dicte » à « je rédige la config en suivant le schéma puis je la fais relire avant déploiement ». Je n'ai pas encore l'autonomie complète sur les configurations BGP complexes, mais je gère seul les déploiements OSPF mono-zone et les ACL standards.
Niveau auto-évalué : Intermédiaire Axes de progrès : maîtriser BGP (peering, route-maps), automatiser les déploiements via Ansible ou Netmiko.

Extrait de configuration OSPF déployée
Configuration OSPF mono-zone que j'ai déployée sur un routeur Cisco ISR (anonymisée). Hardening de base inclus : SSH v2 only, désactivation Telnet et HTTP. 
! Config saisie par Sami AK — relue par ingé Orange
router ospf 1
  network 10.10.0.0 0.0.255.255 area 0
  passive-interface default
  no passive-interface GigabitEthernet0/1
line vty 0 15
  transport input ssh
  exec-timeout 5 0
no ip http server
service password-encryption
Checklist personnelle "Commandes à ne jamais oublier"
Pense-bête que j'ai construit après avoir oublié copy run start et perdu une demi-journée. Désormais collé sur ma sacoche d'intervention. 
# Checklist Sami — avant tout reboot routeur
[1] show running-config # vérif
[2] copy running-config tftp: # backup externe
[3] copy running-config startup-config
[4] show startup-config # confirme
[5] reload # avec confirmation
NOTICE D'INSTALLATION

Fiche Technique — Installation Routeur

Prérequis
Matériel
  • Routeur Cisco ISR 4000 / Huawei AR (selon commande)
  • Câble console RJ45-USB ou RJ45-DB9
  • Modules SFP/SFP+ si interface fibre requise
  • Câbles réseau RJ45 Cat6 (WAN et LAN)
  • Visseaux 19" et clips de baie
Logiciel & Outils
  • PuTTY ou SecureCRT (émulateur console)
  • Fichier de configuration pré-établi par l'ingénieur réseau
  • Schéma d'adressage et plan réseau client
  • Accès GLPI pour clôture d'intervention
Étapes d'installation
1
Réception et inspection matérielle

Vérifier la référence du routeur, les modules installés (interfaces Gig, SFP), l'état physique et le numéro de série. Comparer avec le bon de livraison. Mettre à jour le firmware si une version plus récente est requise.

2
Montage physique en baie 19"

Fixer le routeur dans la baie avec les vis adaptées. Respecter l'espace de ventilation (1U libre au-dessus et en dessous). Relier le câble d'alimentation redondant si disponible. Étiqueter le routeur (hostname, adresse IP management).

Ne pas alimenter l'équipement avant la fin du câblage. Respecter les consignes de sécurité électrique (habilitation B0 H0).
3
Configuration initiale via console

Connecter le câble console (9600 bauds, 8N1). Accéder au mode privilégié (enable). Configurer : hostname, domaine, enable secret (hash SHA-256), utilisateurs locaux, accès SSH v2 uniquement, timeout de session.

4
Configuration des interfaces réseau

Configurer les interfaces WAN (GigE0/0/0 ou port SFP) : adresse IP fixe ou PPPoE. Configurer les interfaces LAN avec sous-interfaces VLAN (encapsulation dot1Q). Activer OSPF ou BGP selon la topologie cible définie dans le schéma réseau.

5
Sécurité et filtrage

Appliquer les ACL entrantes sur l'interface WAN (bloquer les réseaux RFC1918 entrants, les fragments, les spoofing d'adresses). Configurer NAT overload si translation requise. Désactiver les services inutiles (HTTP server, CDP sur WAN, finger…).

Utiliser la commande service password-encryption et stocker la configuration dans un dépôt sécurisé.
6
Intégration supervision SNMP

Configurer SNMP v3 (AuthPriv) avec community string chiffrée. Déclarer l'adresse du serveur NMS Orange (Zabbix/SolarWinds). Tester la collecte OID (sysUpTime, ifOperStatus, bgpPeerState). Activer les traps SNMP pour les alertes critiques.

7
Tests et validation finale

Ping entre tous les VLANs, test traceroute WAN, vérifier les tables de routage (show ip route). Simuler une panne d'interface pour vérifier la convergence OSPF. Sauvegarder la configuration (copy run start).

Checklist de validation
Console SSH accessible (port 22 uniquement)
Table de routage complète et cohérente
Ping inter-VLAN fonctionnel
SNMP v3 collecté par le NMS
Configuration sauvegardée (NVRAM + backup)
Étiquettes et documentation à jour

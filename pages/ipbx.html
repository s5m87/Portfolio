Comprendre l'IPBX

L'IPBX — Téléphonie d'entreprise sur IP

Un IPBX (IP Private Branch eXchange) est un autocommutateur téléphonique privé utilisant le protocole IP pour acheminer les appels. Il remplace l'ancien PABX analogique ou numérique (ISDN) et transforme les communications téléphoniques en flux de données transitant sur le réseau informatique de l'entreprise.

Les solutions les plus déployées chez les clients Orange Business sont 3CX (solution logicielle Windows/Linux), Alcatel-Lucent OXO Connect, Asterisk et la solution cloud Business Talk d'Orange.

  • Gestion des appels internes (internes gratuits entre postes)
  • Trunks SIP vers opérateur pour appels externes
  • Messagerie vocale et messagerie unifiée (email)
  • IVR (serveur vocal interactif) et files d'attente
  • Renvois d'appels, conférences, supervision d'appels
  • Terminaux DECT sans fil pour mobilité interne

Sur un IPBX bien configuré, la qualité voix (MOS) doit être ≥ 4,0 (sur 5). Cela nécessite une QoS réseau avec DSCP EF (Expedited Forwarding) et une latence < 150ms, une gigue < 30ms et un taux de perte < 1%.

Protocoles SIP et RTP

SIP — Session Initiation Protocol

  • Protocole de signalisation (RFC 3261)
  • Port UDP/TCP 5060 (non chiffré)
  • Port TLS 5061 (chiffré et authentifié)
  • Méthodes : INVITE, ACK, BYE, CANCEL
  • Statut : 200 OK, 407 Auth, 404 Not Found

RTP — Real-time Transport Protocol

  • Transport des flux médias (voix/vidéo)
  • Ports UDP dynamiques : 10000–20000
  • SRTP = RTP + chiffrement AES-128
  • RTCP pour statistiques qualité (MOS)
  • Sensible à la latence et la gigue

Codecs voix

  • G.711 PCMA/PCMU : 64 kbps — haute qualité
  • G.729 : 8 kbps — compression forte
  • G.722 : 64 kbps — haute définition (HD)
  • Opus : variable, webRTC moderne
  • Transcoding IPBX si codecs différents

Qualité VoIP (MOS)

  • MOS ≥ 4,0 = qualité excellente
  • Latence max : 150ms (one-way)
  • Gigue max : 30ms
  • Perte paquets max : 1%
  • QoS DSCP EF (46) obligatoire sur réseau

04

Création des extensions et groupements

Créer toutes les extensions internes (postes), groupements d'appels (ring groups) et configurer les renvois.

Extensions et groupements — Asterisk dialplan
! Exemple dialplan Asterisk (/etc/asterisk/extensions.conf) [from-internal] ; Appels internes entre extensions exten => _3XXX,1,Dial(SIP/${EXTEN},30) same => n,VoiceMail(${EXTEN}@default,u) same => n,Hangup() ; Appels vers lignes externes via trunk exten => _0.,1,Dial(SIP/orange-trunk/${EXTEN:1}) same => n,Hangup() ; Groupement accueil (sonnerie simultanée 3010, 3011, 3012) exten => 3000,1,Dial(SIP/3010&SIP/3011&SIP/3012,20) same => n,VoiceMail(3000@default,u) ; Extension avec renvoi vers mobile après 15s exten => 3015,1,Dial(SIP/3015,15) same => n,Dial(SIP/orange-trunk/0612345678) same => n,VoiceMail(3015@default,u)

IVR — Serveur Vocal Interactif

Configurer l'IVR (accueil téléphonique automatique) avec l'arbre d'appels défini par le client.

IVR Asterisk — Accueil + menu
! IVR principal — Numéro d'accueil [ivr-accueil] exten => s,1,Answer() same => n,Wait(1) same => n,Background(accueil-bonjour) ; Message audio : "Bienvenue chez Entreprise X..." same => n,Background(ivr-menu) ; Message : "Tapez 1 pour Commercial, 2 pour SAV..." same => n,WaitExten(5) exten => 1,1,Goto(groupement-commercial,s,1) exten => 2,1,Goto(groupement-sav,s,1) exten => 0,1,Goto(groupement-accueil,s,1) ; Timeout (pas de touche) exten => t,1,Goto(ivr-accueil,s,1) ; Erreur (mauvaise touche) exten => i,1,Goto(ivr-accueil,s,1) ! File d'attente commercial avec musique [groupement-commercial] exten => s,1,Queue(commercial,t,,,120) same => n,VoiceMail(3000@default,u) ; Débordement vers messagerie si attente > 120s
07

Sécurisation TLS / SRTP / fail2ban

Sécuriser l'IPBX contre les attaques SIP (brute-force, SIP flood, toll-fraud) qui peuvent générer des factures téléphoniques très élevées si compromis.

Sécurisation IPBX — Asterisk + fail2ban
! 1. Activer TLS (chiffrement signalisation SIP) ! /etc/asterisk/sip.conf tlsenable=yes tlsbindaddr=0.0.0.0 tlscertfile=/etc/ssl/asterisk/asterisk.pem tlsprivatekey=/etc/ssl/asterisk/asterisk.key tlscafile=/etc/ssl/asterisk/ca.pem ! 2. SRTP obligatoire pour tous les terminaux ! Sur chaque extension encryption=yes avpf=yes ! 3. fail2ban contre brute-force SIP ! /etc/fail2ban/jail.local [asterisk] enabled = true port = 5060,5061 filter = asterisk logpath = /var/log/asterisk/messages maxretry = 3 bantime = 3600 findtime = 600 ! 4. Firewall iptables — limiter SIP iptables -A INPUT -p udp --dport 5060 -m limit --limit 5/sec -j ACCEPT iptables -A INPUT -p udp --dport 5060 -j DROP # Max 5 req SIP/sec depuis même IP ! 5. Whitelist IP du trunk Orange (bloquer tout autre) iptables -A INPUT -p udp --dport 5060 -s IP-TRUNK-ORANGE -j ACCEPT iptables -A INPUT -p udp --dport 5060 -j DROP

Dépannage fréquent IPBX
Trunk SIP désenregistré
Identifiants incorrects, IP source non autorisée chez Orange, ou NAT mal géré
→ sip show registry → Vérifier credentials → Activer nat=force_rport dans sip.conf
Audio dans un seul sens (one-way audio)
Problème NAT media — le flux RTP ne revient pas (adresse privée envoyée à l'opérateur)
→ Activer externip= (IP publique) dans sip.conf → localnet=192.168.0.0/16 → Vérifier ports UDP 10000-20000 ouverts
Qualité voix dégradée (saccades)
Pas de QoS réseau, gigue > 30ms, ou codec G.729 mal implémenté
→ Activer DSCP EF sur switch (mls qos trust dscp) → Passer en G.711 → Mesurer gigue avec Wireshark
Toll fraud (appels frauduleux)
IPBX exposé sur internet sans protection, extensions avec mots de passe faibles
→ fail2ban immédiatement → Changer tous les MDP → Whitelist IP trunk Orange → Surveiller CDR appels

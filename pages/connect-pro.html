Présentation de l'offre

Qu'est-ce que Connect Pro ?

Connect Pro est la solution d'accès internet professionnel d'Orange dédiée aux entreprises de toute taille. Elle garantit une connectivité haut débit avec un engagement de disponibilité contractuel (SLA), une qualité de service (QoS) administrée et un support technique prioritaire 24h/24.

Contrairement à une offre grand public, Connect Pro intègre :

  • Un accès dédié ou mutualisé selon le contrat (débit garanti jusqu'à 100%)
  • Une adresse IP fixe (IPv4 et/ou IPv6)
  • Un SLA de disponibilité jusqu'à 99,9% avec pénalités contractuelles
  • Un équipement CPE fourni et maintenu par Orange
  • Un portail d'administration Business permettant la supervision en temps réel
  • Option de backup 4G/LTE en cas de coupure (failover automatique)

Notice d'installation — Étape par étape
01

Accueil et vérification documentaire

À l'arrivée chez le client, se présenter et vérifier que les informations du bon de commande correspondent à la prestation attendue. Confirmer l'emplacement de l'installation.

Points à vérifier
✓ Numéro de référence commande
✓ Adresse du site correspond
✓ Type d'offre (débit, technologie d'accès)
✓ Adresse IP fixe attribuée par Orange
✓ Accès à la baie ou au local technique
02

Raccordement physique du CPE

Installer le modem/routeur CPE dans la baie ou l'espace désigné. Connecter le câble d'accès WAN (ligne DSL ou fibre) et les câbles LAN vers le réseau client.

Câblage standard
PORT WAN → Prise téléphonique / PTO fibre
PORT LAN1 → Switch principal client (ou PC direct)
PORT LAN2 → Backup / DMZ si besoin
Alimentation → Bloc secteur 230V stabilisé

⚠ Vérifier la polarité sur ligne DSL (L1/L2)
✓ LED WAN doit devenir verte après sync
03

Configuration de l'interface WAN (PPPoE)

Accéder à l'interface d'administration du CPE et configurer la connexion WAN avec les identifiants PPPoE fournis dans la fiche technique Orange.

Interface d'admin CPE (ex: Cisco / Huawei)
interface dialer 0
 encapsulation ppp
 ppp chap hostname login@orange.fr
 ppp chap password motdepasse
 ip address negotiated
 no shutdown

ip route 0.0.0.0 0.0.0.0 dialer0
ip nat outside → sur interface WAN
! IP fixe attribuée automatiquement via PPPoE
04

Configuration du réseau local (LAN / DHCP / NAT)

Configurer l'interface LAN avec l'adressage IP du client, activer le serveur DHCP et mettre en place le NAT/PAT pour la translation d'adresses.

Configuration LAN + DHCP + NAT
! Interface LAN
interface GigabitEthernet0/1
 ip address 192.168.1.1 255.255.255.0
 ip nat inside
 no shutdown

! Pool DHCP
ip dhcp pool CLIENT_LAN
 network 192.168.1.0 255.255.255.0
 default-router 192.168.1.1
 dns-server 80.10.246.2 80.10.246.129
 lease 1

! NAT overload (PAT)
ip nat inside source list 1 interface Dialer0 overload
access-list 1 permit 192.168.1.0 0.0.0.255
05

Configuration VLAN (si segmentation requise)

Si le client dispose de plusieurs services à isoler (Voix, Données, Management), configurer les VLANs sur le CPE et le switch en aval.

Exemple VLAN 802.1Q — 3 segments
! Sous-interfaces LAN avec VLAN
interface Gi0/1.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
 description VLAN-DATA

interface Gi0/1.20
 encapsulation dot1Q 20
 ip address 10.0.20.1 255.255.255.0
 description VLAN-VOIX
 ip dscp ef <!-- Priorité QoS VoIP -->

interface Gi0/1.99
 encapsulation dot1Q 99
 ip address 172.16.99.1 255.255.255.0
 description VLAN-MGMT
06

Activation et tests de connectivité

Vérifier la synchronisation WAN, tester la connectivité internet et mesurer les débits dans les deux sens pour valider le SLA.

Commandes de validation
! Vérifier l'état de l'interface WAN
show interface Dialer0
→ Dialer0 is up, line protocol is up
→ Internet address is 90.x.x.x/32

! Test connectivité internet
ping 8.8.8.8 repeat 100
→ !!!!! (100% success, <5ms)

! Vérification route par défaut
show ip route 0.0.0.0

! Test débit (depuis PC client)
speedtest-cli --server orange → ↓ 200 Mbps ↑ 200 Mbps
07

Test de bascule Failover 4G

Si l'option backup 4G est souscrite, tester la bascule automatique en simulant une coupure de la ligne principale.

Test failover
! Déclencher la bascule manuellement
interface Dialer0
 shutdown

! Vérifier que le backup prend le relais
show ip interface brief
→ Cellular0/0/0 — UP — IP 4G attribuée
→ Ping 8.8.8.8 répond via 4G

! Restaurer la ligne principale
interface Dialer0
 no shutdown
→ Bascule retour automatique en <30s
08

Clôture et remise en main client

Rédiger le compte-rendu d'intervention, noter les paramètres de configuration et remettre les accès au client avec explication.

Éléments à documenter (CR d'intervention GLPI)
✓ Date, heure début/fin d'intervention
✓ Référence commande et numéro de ticket
✓ Adresse IP WAN fixe attribuée
✓ Adresse IP LAN du routeur (passerelle)
✓ Plage DHCP configurée
✓ VLANs créés (ID + description + subnet)
✓ Résultat tests de débit (↓ X Mbps / ↑ X Mbps)
✓ Signature du bon de recette client
✓ Photos de l'installation câblage
Dépannage fréquent
LED WAN rouge / pas de sync
Ligne non activée côté NRO Orange, ou câble DSL défectueux
→ Vérifier activation dans le SI Orange → Tester avec câble de remplacement → Contacter le support Orange si persistant
Authentification PPPoE échoue
Identifiants PPPoE incorrects ou ligne non provisionnée
→ Vérifier login/password PPPoE dans la fiche technique → Confirmer le provisionnement auprès du back-office Orange
Pas d'accès internet côté LAN
NAT non configuré, route par défaut manquante
→ Vérifier show ip nat translations → Vérifier show ip route 0.0.0.0 → Contrôler access-list NAT
Débit inférieur au contrat
Atténuation élevée sur ligne DSL, ou QoS mal configurée
→ Mesurer l'atténuation ligne (ATUC/ATUR) → Vérifier la distance au NRO → Signaler au NOC Orange si débit garanti non atteint
